
# 餐损

- **判断订单状态**
  - **条件: 订单已取消**
    - **判断配送方式**
      - **条件: 配送服务2.0**
        - **根据“餐损赔付记录”或“餐损状态”判断**
          - **餐损赔付记录：定责原因=天气** -> 安抚天气问题
          - **餐损赔付记录：定责原因=未知** -> 走“取消方判断”流程
          - **餐损状态：待定责** -> 判断取消时间是否超过48h
            - **是**: 走“取消方判断”流程
            - **否**: 餐损审核中等待
          - **餐损状态：骑手责任直赔 / 美团责任直赔** -> 餐损自动赔付
          - **餐损状态：美团责任完成率** -> 判断当前日期是否早于下单日的次周四
            - **是**: 告知等待完成率结果
            - **否**: 告知以完成率结果为准
          - **餐损状态：用户责任不可申诉 / 商家责任不可申诉 / 申诉超时 / 二次申诉商家责任 / 无餐损标签+取消时间超过48h** -> 走“取消方判断”流程
          - **餐损状态：商家责任可申诉 / 骑手责任可申诉** -> 告知商家端上申诉
          - **餐损状态：二次申诉审核中** -> 餐损审核申诉中等待
          - **餐损状态：二次申诉用户责任** -> 走“取消方判断”流程

      - **条件: 跑腿and企客**
        - **根据“餐损状态”判断**
          - **已取消可发起餐损申请** -> 自行申请餐损
          - **待审核/定责中/已申请待处理** -> 餐损审核中等待
          - **一审通过** -> 餐损自动赔付
          - **一审驳回可申诉** -> 告知商家端上申诉
          - **一审驳回不可申诉 / 二审驳回 / 定责完成不可申诉 / 申诉驳回** -> 走“取消方判断”流程
          - **申诉待审核** -> 餐损审核申诉中等待
          - **申诉通过** -> 餐损自动赔付

      - **条件: 城市代理**
        - **根据“餐损状态”或“无餐损标签”的取消原因判断**
          - **(与“跑腿and企客”流程类似，包含各种状态的处理)**
          - **无餐损标签+客服超72h未及时处理用户退款导致订单自动退款** -> 走“取消方判断”流程
          - **无餐损标签+超预计送达30min导致订单退款** -> 走“取消方判断”流程

      - **条件: 商家自配**
        - **未挂载，维持原状**

      - **条件: 聚和配送**
        - **取消操作方=AIDA or TASK AC or人工客服 and 取消责任方=平台** -> 餐损手动赔付
        - **非以上情况** -> 引导联系站长/BD

  - **条件: 订单部分退款or未退款但商家重复出餐**
    - **判断配送方式**
      - **配送服务2.0 / 跑腿and企客** -> 无法申请餐损
      - **城市代理 / 聚和配送** -> 建议商家联系站长/BD
      - **商家自配** -> 未挂载，维持原状

### **取消方判断 (通用子流程)**

- **根据取消方和取消原因进行最终判责**
  - **取消方: [系统因子]商家取消**
    - **判断售后或履约场景**
      - **售后场景**: 无法申请餐损
      - **履约场景**: 判责 (商家:无法申请; 用户/非商非用户:手动赔付)
  - **取消方: [系统因子]系统取消 (多种原因)**
    - **无法申请餐损**
  - **取消方: [系统因子]客服取消 = 商家责任**
    - **判断是否拼好饭**
      - **是 (拼好饭)**: 判断售后或履约场景 -> 最终判责
      - **否 (非拼好饭)**: 判断售后或履约场景 -> 最终判责
  - **取消方: [系统因子]客服取消 = 非商家责任**
    - **判断售后或履约场景 -> 最终判责**
  - **取消方: [系统因子]系统取消 and [系统因子]取消原因(...)**
    - **判断售后或履约场景 -> 最终判责**
